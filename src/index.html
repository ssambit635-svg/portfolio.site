How I Work

From idea to shipped.

The way I actually approach building things.

01

Figure out what I'm actually building

Before writing a single line — what's the problem, what's the output, what does "done" look like. Saves hours of going in circles.

02

Build the core, break it, fix it

Get the main thing working first — ugly, messy, functional. Then clean it up. I learn more from breaking things than reading about them.

03

Ship it and keep improving

Deploy early. Real feedback from a live URL is worth more than perfecting it locally. Then keep updating as I learn more.
